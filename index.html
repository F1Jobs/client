FAQs


1. What is the purpose of this site?

If you are looking for a job at one of the Formula 1 teams, browsing their individual pages for job listings gets tedious. So I made this site to display it all in one place, and apply for various positions easily.


2. Tell me more.

The point of this website is simple - To keep an eye on the job listings of the F1 teams currently on the grid without spending much effort. I'm surprised how many sites won't let you view just the job titles without having to wade through a swamp of JavaScript. I wanted to view them all in one place, and initally wrote a CLI version for myself. Later I converted it into this website.


3. How frequently does it get updated?

Any update to job listings at original sites gets updated here within 30 minutes. Job listings don't change dramatically in a short period of time, so hitting the original sites for instantaneous updates would only make this site slower.


4. Fast, you say. How so?

  • This is a static website. It uses the Fetch API to get stored data (which is updated periodically), as opposed to gathering it at load time.
  • It is hosted on Github Pages, which, rather than serving the content directly, serves it using a global Content Delivery Network.
  • No external JS libraries are loaded. In fact, there are none involved. Also, there are no scripts for tracking or analytics.


5. Can I see the source?

Sure, the website is open-source under the GNU General Public License v3.0. The source code is hosted on GitHub.
